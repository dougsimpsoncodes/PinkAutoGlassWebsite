# Google Ads Negative Keywords - Rules & Best Practices

**Created:** November 4, 2025
**Purpose:** Prevent mistakes when creating negative keyword lists

---

## THE FUNDAMENTAL RULE

**NEGATIVE KEYWORDS = BE AGGRESSIVE**
**POSITIVE KEYWORDS = BE CONSERVATIVE**

These are opposites. Never confuse them.

---

## Critical Mistake Made (November 4, 2025)

### What Happened:
When creating location-based negative keywords, I initially suggested using only "City State" format (e.g., "Divide Colorado") and warned against adding standalone city names like "Divide" because it might block phrases like "divide costs" or "Monument Valley."

### Why This Was Wrong:
I applied **positive keyword logic** to **negative keywords**. For negative keywords:
- ✅ Blocking "divide costs" is GOOD (not your customer)
- ✅ Blocking "Monument Valley" is GOOD (wrong location)
- ✅ There is NO DOWNSIDE to being aggressive
- ✅ The goal is to eliminate ALL irrelevant traffic

### The Correct Approach:
Be as aggressive as possible. Add both:
- "Divide Colorado" (specific)
- "Divide" (catches more variations)

## Decision Framework: Positive vs Negative Keywords

### POSITIVE KEYWORDS (Adding to Campaign)
**Goal:** Attract customers, don't miss opportunities

**Be Conservative:**
- ❌ Don't add broad terms that might waste budget
- ❌ Don't add terms with ambiguous intent
- ✅ Use exact match or phrase match for control
- ✅ Start narrow, expand based on performance
- ✅ Consider: "Could this person NOT be my customer?"

**Example:**
- Add: `[windshield replacement denver]` (exact match)
- Add: `"cheapest windshield near me"` (phrase match)
- Don't add: `glass` (too broad, could be windows, mirrors, drinking glasses)

---

### NEGATIVE KEYWORDS (Blocking Traffic)
**Goal:** Eliminate waste, block irrelevant traffic

**Be Aggressive:**
- ✅ Block anything that's not your customer
- ✅ Block even if there's a tiny chance of ambiguity
- ✅ Use broad match or phrase match liberally
- ✅ When in doubt, BLOCK IT
- ✅ Consider: "Could this person waste my budget?"

**Example:**
- Block: `"divide"` (even though it's a common word)
- Block: `"monument"` (even though it's a common word)
- Block: `"jobs"` (any job seeker is irrelevant)
- Block: `"how much"` (informational, not transactional)

---

## Why Aggressive Negative Keywords Are Safe

### 1. Zero Downside
- If you block "divide costs", the person wasn't going to buy windshield service anyway
- Even if someone searches "how to divide windshield cost between insurance and me" and you block them, they're probably not ready to buy

### 2. Positive Keywords Trump Negatives
- If you have a positive keyword `[windshield replacement]`
- And negative keyword `"divide"`
- A search for "windshield replacement" will still trigger your ad
- The negative only blocks if BOTH match

### 3. You Can Always Remove Later
- If you accidentally block something valuable (rare), you'll see it in reports
- Easy to remove from negative list
- Better to block first, unblock later than waste money

## Location-Based Negative Keywords: Specific Rules

### Rule 1: Always Use Multiple Formats

For each location to block, add ALL of these:

```
"City State"        → "Divide Colorado"
"City"              → "Divide"
"City, State"       → "Divide, Colorado"  (if needed)
```

### Rule 2: Don't Worry About Common Words

Even if the city name is a common word, block it:

**Cities that are common words:**
- Divide
- Monument
- Victor (name)
- Grover (name)
- Superior
- Commerce City

**Why it's safe to block:**
- People searching for auto glass won't use these terms randomly
- If they do, they're not high-intent anyway
- The cost of being too conservative is wasted ad spend

### Rule 3: Block Tiny Towns Aggressively

Any town with population < 5,000:
- ✅ Block it (even if it got a click)
- ✅ Block it (even if CPC was low)
- ✅ Only exception: If it CONVERTED

## Competitor Name Negative Keywords: Specific Rules

### Rule 1: Block Variations Liberally

For competitor "Safelite":
```
[safelite]
[safelight]
[savelite]
[safe lite]
"safelite auto glass"
"safelite solutions"
```

### Rule 2: Use Phrase Match for Generic Competitors

For "auto glass now":
```
"auto glass now"     → Blocks: "auto glass now denver", "call auto glass now"
"glass now"          → Blocks even shortened versions
```

### Rule 3: Don't Worry About Blocking Your Own Brand

If your name is "Pink Auto Glass":
- Someone searching "pink auto" is probably looking for you
- But if you accidentally block "pink" as a negative, your brand searches will still work
- Brand keywords have strong positive signals

## Job Seeker Negative Keywords: Specific Rules

### Always Block These Terms (Broad Match)

```
jobs
careers
hiring
employment
apply
work
resume
technician jobs
installer jobs
glass technician
```

**Why:** Zero chance a job seeker becomes a customer

## Informational Query Negative Keywords

### Block These Aggressively

```
"how much"
"how to"
"cost of"
"estimate"
"average cost"
"typical cost"
"where to buy"
"who fixes"
"can you fix"
"diy"
```

**Why:** These are research queries, not buying queries

**Exception:** If you have a blog strategy, create separate campaign for informational content with much lower bids

## Quick Decision Checklist

When deciding whether to add a negative keyword, ask:

### 1. Could this person be my customer?
- ✅ **NO** → Add to negative list
- ⚠️ **MAYBE** → Add to negative list (be aggressive)
- ❌ **YES** → Don't add

### 2. Is this a competitor's brand?
- ✅ **YES** → Add all variations to negative list
- ❌ **NO** → Evaluate further

### 3. Is this a wrong location?
- ✅ **YES** → Add city name + city+state to negative list
- ❌ **NO** → Keep

### 4. Is this informational intent?
- ✅ **YES** → Add "how to", "cost of" patterns to negative list
- ❌ **NO** → Keep

### 5. Has this term converted?
- ✅ **YES** → DO NOT ADD to negative list (even if it seems weird)
- ❌ **NO** → Evaluate further

### 6. When in doubt?
→ **ADD IT TO NEGATIVE LIST**
→ You can always remove it later if needed

## The Golden Rule

**FOR NEGATIVE KEYWORDS:**

> When in doubt, block it.
>
> The cost of blocking a good query by accident: $0 (you just don't show the ad)
>
> The cost of not blocking a bad query: $10-40+ per click with 0% conversion

**Risk Assessment:**
- Accidentally blocking good traffic: Low risk, easy to fix
- Not blocking bad traffic: High cost, ongoing waste

**Therefore:** Always err on the side of blocking.

---

## Monthly Review Process

1. Export search terms report
2. Look for:
   - New competitor names appearing
   - New out-of-area locations getting impressions
   - Informational queries getting clicks
   - High CPC terms with 0 conversions
3. Add ALL of them to negative list (be aggressive)
4. Only remove negatives if you see evidence you blocked something valuable

---

## Red Flags That Mean "Block Immediately"

- 🚩 Any competitor brand name
- 🚩 Any location with < 5,000 population
- 🚩 Any CPC > $30 with 0 conversions
- 🚩 Any term with "job", "career", "hiring"
- 🚩 Any term with "how to", "diy", "yourself"
- 🚩 Any term with "wholesale", "used", "for sale"
- 🚩 Any out-of-state location
- 🚩 Any query pattern seen 3+ times with 0 conversions

---

## Key Takeaway

The mistake made on November 4, 2025 was thinking like we were adding POSITIVE keywords when we were creating NEGATIVE keywords.

**Remember:**
- Positive keywords = Be careful (might waste money)
- Negative keywords = Be aggressive (saves money)

**Never confuse the two again.**
